Implemented the rate-limited JSON API in the Qwik City project at `/home/user/qwik-app`:

**`src/lib/rate-limiter.ts`** — shared, in-memory, per-IP sliding-window limiter (5 req / rolling 5s), exposing:
- `consume(ip)` — records+counts a request (mutates state), returns `{ allowed, remaining, limit, resetSeconds, retryAfterSeconds }`.
- `peek(ip)` — read-only lookup, prunes expired entries but never increments the count.
- `getClientIp(request, connIp)` — resolves IP from first entry of `X-Forwarded-For`, falling back to the connection IP.

**`src/routes/api/layout.ts`** — shared `onRequest` middleware for everything under `/api/`:
- Computes the client IP, calls `consume()` exactly once per request, stores the result in `sharedMap`.
- Always sets `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `X-RateLimit-Reset`.
- When blocked, sets `Retry-After` and `throw`s `requestEvent.json(429, {...})`, which halts the chain so `/api/data`'s handlers never execute.

**`src/routes/api/data/index.ts`** — `onGet`/`onPost` handlers returning JSON `{ method, message, remaining, limit }` with status 200 (reusing the rate-limit result already computed by the middleware via `sharedMap`, so quota isn't double-counted).

**`src/routes/status/index.tsx`** — non-`/api` HTML page using a `routeLoader$` that calls `peek()` (never `consume()`) for the requesting IP, rendering `Limit: 5`, `Remaining: <n>`, `Reset: <n>s` as single contiguous text nodes (to avoid Qwik's resumability comment markers splitting the label from the value in the raw HTML).
