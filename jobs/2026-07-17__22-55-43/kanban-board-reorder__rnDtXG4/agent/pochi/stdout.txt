The board has three fixed columns — `todo`, `doing`, and `done` — each holding an ordered list of cards. Users move cards between columns and reorder them within a column using native HTML5 drag-and-drop. Every move is persisted to a local SQLite database so that the exact column and ordering survive a full page reload. The board is server-rendered from the database on every request. ## Requirements * Render a board page at `/` with three columns (`todo`, `doing`, `done`), each showing its cards in persisted order. * Cards must be movable between columns and reorderable within a column via native HTML5 drag-and-drop (wired with `useStore` and `onDragStart$` / `onDragOver$` / `onDrop$`). * Persist every move to a local SQLite database, maintaining a stable, contiguous 0-based `position` index within each column. * Server-render the board from the database using `routeLoader$` so ordering is correct after a reload (resumability / rehydration). * Expose JSON HTTP endpoints for reading the board, applying a move, and resetting to the initial seed. * Seed the database with the initial board state below when it is empty. ## Data model Each card has: a stable integer `id`, a `title` (string), a `column` (`todo` | `doing` | `done`), and a `position` (0-based integer, contiguous within its column, ascending = top-to-bottom).
